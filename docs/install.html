The N.A. Cleantime Calculator (NACC)

Installing NACC

NACC is very easy to install. It has been designed as a standalone package. It requires including one JavaScript file, and two CSS files into the <head></head> section of your Web page. You invoke it by calling one JavaScript function.

NACC has also been designed as a WordPress plugin. If you have a WordPress blog, NACC will easily fit into any page or post that you desire.

First, get the code. The current version is available by going to this page, and downloading the latest version of the NACC: http://sourceforge.net/project/showfiles.php?group_id=209495&package_id=251359

You start by including the entire nacc directory into your Web server files. Here is a manifest of the NACC directory:

 

NACC Directory Manifest:


  • changelog.txt
    This is a file that explains the current version, and changes made to the NACC.
  • demo.html
    This is a standalone demonstration of the NACC, using the "BT" theme.
  • docs (directory)
    This contains this documentation.
  • images (directory)
    This contains all the images used in the NACC.
  • nacc.css
    This is the principal CSS file for the NACC. It needs to be installed, regardless of what theme is used.
  • nacc.js
    This is the "heart" of the NACC. It is an extensively-documented JavaScript file that holds all of the NACC code.
  • nacc.php
    This is the WordPress plugin file. It has been written using the "BT" theme.

    The following three files implement our included "themes":

  • nacc_theme_bt.css
    This uses the "Basic Text" "blue and white"color theme.
  • nacc_theme_hw.css
    This uses the "It Works -How and Why" "green and gold" theme.
  • nacc_theme_jft.css
    This uses the "Just For Today" "red and gray" theme colors.

 

Installing NACC in a regular HTML-based Web Site:


You'll need to edit the <head> section of your file. For this example, we'll assume that the nacc directory is at the same level as the file that will implement it:

<html>
   <head>
      <title>NACC Test</title>
      <script type="text/javascript" src="nacc/nacc.js"></script>
      <link rel="stylesheet" href="nacc/nacc.css" type="text/css" />
      <link rel="stylesheet" href="nacc/nacc_theme_bt.css" type="text/css" />
   </head>
   <body>
     •
     •
     •
   </body>
</html>

Note the highlighted "nacc/nacc_theme_bt.css". This is the theme CSS file. You can exchange it for either of the other two we've provided ("nacc/nacc_theme_hw.css" or "nacc/nacc_theme_jft.css"). You could also write your own, or modify one of the default themes. Including another theme after the one provided will override it.

The only other steps are to provide a "container" <div> element, and then to invoke the NACC. These are done in the <body> section of your file:

<html>
      <head>
        •
        •
        •
      </head>
      <body onload="NACC_CleanTime('cleantime_calculator','nacc')">
         <div id="cleantime_calculator"></div>
      </body>
</html>

We are invoking it with all the options (explained below) as default (except the path). The only required parameter to the NACC_CleanTime() function is the DOM ID of the element that will contain it.

Because the NACC is JavaScript-based, you should put a <noscript> section into the "container." This gives a message to visitors that don't have JavaScript enabled:

<html>
      <head>
        •
        •
        •
      </head>
      <body onload="NACC_CleanTime('cleantime_calculator','nacc')">
         <div id="cleantime_calculator"><noscript><h1>JavaScript Not Enabled</h1><h2>Unfortunately, the cleantime calculator relies on JavaScript, and your browser does not support JavaScript.</h2></noscript></div>
      </body>
</html>

That leaves us with the following as a complete implementation of the NACC in the "Basic Text" theme:

<html>
   <head>
      <title>NACC Test</title>
      <script type="text/javascript" src="nacc/nacc.js"></script>
      <link rel="stylesheet" href="nacc/nacc.css" type="text/css" />
      <link rel="stylesheet" href="nacc/nacc_theme_bt.css" type="text/css" />
   </head>
   <body onload="NACC_CleanTime('cleantime_calculator','nacc')">
      <div id="cleantime_calculator"><noscript><h1>JavaScript Not Enabled</h1><h2>Unfortunately, the cleantime calculator relies on JavaScript, and your browser does not support JavaScript.</h2></noscript></div>
   </body>
</html>

 

Installing NACC in a WordPress Blog:


You'll need to start by placing the nacc directory into the wp-content/plugins/ directory. Go into the "plugins" pane of the admin interface, and activate "nacc". This will give you the default "BT" theme, with cookies enabled (saves the user's cleantime). You can edit the "nacc.php" file to make adjustments to the included theme file and the options.

In order to invoke the NACC in a page or a post, you need to add the following content (in the "code" pane of the editor):

<!-- NACC -->

This comment will be replaced by the cleantime calculator. You don't need to do anything else, and the comment can be mixed in with a bunch of other text. Just remember that calculating the cleantime can result in a great many tags being displayed, so text under it may be a long vertical scroll down.

 

NACC Options:


There are a number of optional arguments to the function call that invokes the NACC:

 

  • pContainerId
    This is the only required parameter. This must be the DOM ID of the containing HTML element.
  • pPathToNACC
    This is the path to the main NACC directory (the "nacc" directory). If the code is actually inside the nacc directory, this can be left blank. The path is relative to the file that is implementing the NACC.
  • pUseCookie
    Set this to true if you want to use a cookie to remember the visitor's cleantime and layout preference. Default is off.
  • pChained
    Set this to true if you want the default layout to be a long, vertical chain. Otherwise, the default layout will be "tabletop," which is a horizontal row of overlapping keytags.
  • pHideTags
    The tags are fairly substantial files. In some cases, you may not want to burden your visitors with having to download them. In this case, set this to true. Otherwise, the keytags will always be displayed.


The calling syntax for the NACC function is:

NACC_CleanTime(pContainerId, pPathToNACC, pUseCookie, pChained, pHideTags);

Hopefully, this will explain how to use the NACC to your satisfaction. If you want to know more, the nacc.js file is heavily documented.

 

NEW IN 2.0: Persian (Iranian) Calendar Support


The Iranians (and Afghani) use a different calendar system than Western nations. It is called the "Solar" or Shamsi Calendar. We have added support for this. Thanks to NA Iran for helping us to test and localize it.

In order to use this, you need to set the gCalendarType variable in nacc.js to "Persian" (It is "Gregorian" by default).

 

NEW IN 1.7: DECADES CLEAN (PURPLE) TAG


We have added a new, optional keytag (Purple "Decades" tag, that will show every ten years after ten (20, 30, etc.).

In order to enable this, set the gUseDecadesTag variable in nacc.js to true (It is false by default).

 

A NOTE ABOUT THE FIPT:


The symbol used in the keytags is the registered N.A. symbol. It should only be displayed by duly authorized official N.A. Web sites (sites for official N.A. Service bodies, like Regions, Zonal Forums, Metro Areas, Area Service Committees or Groups).

Official N.A. Service symbols are covered by the FIPT (Fellowship Intellectual Property Trust), and must be treated respectfully and properly at all times.

In the general version of the NACC, we provide "generic" keytags that do not have the official N.A. symbol on them.
